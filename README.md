## 유의사항

비전 AI 개발 특성 상 여기 사용될 의존성은 클라우드 환경에서 돌리기 불가능하므로 아래와 같이 진행
- AI 관련 기능은 로컬에서만 테스트 → 테스트 및 개발 완료 후 로컬 MS로 분리
- 로컬 MS는 테스트, 파인튜닝, 이미지 생성 등을 담당
- 로컬 MS에서 생성한 이미지를 GCP 스토리지 버킷에 업로드
- 프론트엔드에서 GCP 스토리지 버킷에 업로드된 이미지를 읽는 방식으로 개발

## 의존성 설치 가이드

### 1. 특정 그룹(ai)만 포함해서 설치할 때
```shell
uv pip install -e . --extra ai
```

### 2. 만약 모든 그룹(dev, ai 등)을 한꺼번에 설치하고 싶을 때 (로컬 개발 환경)
```shell
uv sync --all-groups
```
